# `--edit` Flag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `--edit` flag to all four `run-*.sh` launchers that resolves+seeds the config dir the script would mount, opens it in the user's editor, and exits without launching a container.

**Architecture:** Each script gets (1) a pre-`getopts` argv scan that extracts `--edit` (stopping at `--` so agent passthrough is untouched) and (2) an action block, placed after the existing resolve+seed logic and before the container section, that `exec`s an editor on the resolved host config dir. Editor precedence: `$VISUAL` → `$EDITOR` → `nvim` → `vi`.

**Tech Stack:** Bash (`getopts`, `exec`); editors `nvim`/`vi`. No test framework — verification is bash harness scripts with a stub editor.

---

## File structure

Four launcher scripts get the identical two-part change (different config-dir variable + help `sed` range each). README gets doc updates.

- `run-claude.sh` — config var `CONFIG_DIR`; help range `'2,28p'` → `'2,29p'`. Has `SCRIPT_DIR`.
- `run-pi.sh` — config var `CONFIG_SRC`; help range `'2,25p'` → `'2,26p'`. Has `SCRIPT_DIR` + bundle seed.
- `run-goose.sh` — config var `CONFIG_SRC`; help range `'2,24p'` → `'2,25p'`. Has a `set -- session` no-args block.
- `run-hermes.sh` — config var `CONFIG_SRC`; help range `'2,24p'` → `'2,25p'`.
- `README.md` — flag table + Run-section mention.

Placement rules (same for all):
- **Scan**: immediately before the `while getopts ... do` loop.
- **Action block**: after the line that finalizes the config dir (and after any seeding), before the `if [ -n "$NAME" ]` named-container section. In `run-goose.sh` it goes before the `# No args -> ... session` block too.

- [ ] **Step 4: Bump the help `sed` range**

Change `h) sed -n '2,28p' "$0"; exit 0 ;;` to `h) sed -n '2,29p' "$0"; exit 0 ;;`
(the added usage line shifts the examples down by one).

- [ ] **Step 5: Add the pre-getopts scan**

Immediately before `while getopts "c:iHw:n:h" opt; do`, insert:

```bash
# Extract the long flag --edit before getopts (which only handles short opts).
# Stop at `--` so agent passthrough args keep their own --edit, if any.
EDIT=0
_args=(); _stop=0
for _a in "$@"; do
  [ "$_stop" -eq 0 ] && [ "$_a" = "--" ] && _stop=1
  if [ "$_stop" -eq 0 ] && [ "$_a" = "--edit" ]; then EDIT=1; continue; fi
  _args+=("$_a")
done
set -- "${_args[@]}"
```

- [ ] **Step 6: Add the action block**

After the line `CONFIG_JSON="${CONFIG_DIR}.json"; touch "$CONFIG_JSON"`, insert:

```bash
# --edit: open the resolved host config dir in an editor, then exit (no container).
if [ "$EDIT" -eq 1 ]; then
  ED="${VISUAL:-${EDITOR:-}}"
  [ -z "$ED" ] && { command -v nvim >/dev/null 2>&1 && ED=nvim || ED=vi; }
  exec $ED "$CONFIG_DIR"
fi
```

- [ ] **Step 7: Run test to verify it passes**

Run: `bash /tmp/test-edit-claude.sh`
Expected: `PASS`

- [ ] **Step 8: Syntax check**

Run: `bash -n run-claude.sh && echo OK`
Expected: `OK`

- [ ] **Step 9: Commit**

```bash
git add run-claude.sh
git commit -m "feat(run-claude): add --edit to open config dir in editor"
```
